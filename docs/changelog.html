Changelog

All notable changes to CAD Preprocess are documented here.

The format is based on Keep a Changelog, and this project adheres to Semantic Versioning.

[0.1.0] - 2026-01-10

ðŸŽ‰ Initial Release

Added

  • Input Handler Module
    • Recursive DICOM file discovery
    • DICOM validation and filtering
    • Support for .dcm, .dicom, and extensionless files
  • Preprocessing Engine
    • Deterministic image preprocessing pipeline
    • CT windowing with configurable center/width
    • Rescale slope/intercept application
    • Image normalization to 0-255
    • Configurable resize with multiple interpolation methods
  • Metadata Extractor
    • Profile-based metadata extraction
    • Profiles: minimal, patient, geometry, ml, acquisition, full
    • Custom tag extraction support
  • Output Writer
    • PNG, JPEG, and NumPy output formats
    • Configurable naming policies (sop_uid, patient_study, sequential)
    • Automatic directory structure creation
    • Processing manifest generation
  • Configuration System
    • YAML configuration file support
    • Environment variable overrides
    • CLI argument overrides
    • Configuration hashing for reproducibility
  • CLI Interface
    • Full-featured command-line interface
    • Dry-run mode for previewing
    • Verbose output option
  • Logging & Error Handling
    • Structured JSON logging
    • Processing statistics tracking
    • Fail-safe decorator for graceful error handling
  • Integration API
    • preprocess() one-liner function
    • CADPreprocessor class for batch processing
    • Config hash for train/inference reproducibility

Dependencies

  • Python >= 3.9
  • pydicom >= 2.3.0
  • numpy >= 1.21.0
  • Pillow >= 9.0.0
  • PyYAML >= 6.0
  • scikit-image >= 0.19.0
